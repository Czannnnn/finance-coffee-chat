# DART document.xml 응답 구조 분석 (P1-B 단계 2)

**조사 일자:** 2026-04-26
**목적:** P1-B 단계 2 — `fetchDartDocDetail_()` 구현을 위한 응답 포맷·필드 위치·정규식 후보 확정

## 1. API 호출 사양

```
GET https://opendart.fss.or.kr/api/document.xml
    ?crtfc_key={DART_API_KEY}
    &rcept_no={14자리 접수번호}
```

- **응답 1**: ZIP 파일 (PK\x03\x04 헤더). 안에 `{rcept_no}.xml` 단일 파일.
- **응답 2 (데이터 없음)**: status=014 의 짧은 XML. 예: SPAC(`대신밸런스제20호기업인수목적`) 같은 일부 공시는 본문 비공개.
- 인코딩: UTF-8 고정 (XML 헤더 `encoding="utf-8"`)
- 크기: 800KB ~ 4MB (회사 규모/정정 횟수에 따라)
- 루트: `<DOCUMENT>` → `<DOCUMENT-NAME>` + `<COMPANY-NAME>` + `<BODY>`. 본문은 거대한 `<TABLE>` 중첩 + `<TR>`/`<TD>`/`<P>`/`<SPAN>`. 사실상 HTML과 유사.

## 3. 추출 가능 필드 (P1-B 적용 대상)

### 3-1. 확정공모가 (`confirmed_price`)

- **출현 조건**: `[발행조건확정]` 공시에서만 명시. `[기재정정]`/`[정정]` 단계에서는 "공모가액 확정 후 정정신고서를 제출할 예정"이라는 미래형 문구만.
- **신호 문구 (코스모로보틱스 실측)**:
  > "공동대표주관회사와 발행회사인 코스모로보틱스㈜가 협의하여 1주당 확정공모가액을 **6,000원**으로 최종 결정하였습니다."

- **정규식 후보 (우선순위 순)**:
  ```
  /확정\s*공모\s*가액[을이]?\s*([\d,]+)\s*원\s*으로\s*(?:최종\s*)?결정/
  /1\s*주당\s*확정\s*공모\s*가액[을이]?\s*([\d,]+)\s*원/
  /확정\s*공모\s*가액[은인]\s*([\d,]+)\s*원/
  ```
- **추출 규칙**: 위 정규식 첫 매치 → `parsePrice_()` 적용. 매치 실패 시 `null` 반환.
- **함정**: 같은 XML 안에 "확정공모가액"이 60+회 등장 (정의 설명·의무보유확약 단가 기준 등). "최종 결정" 키워드 포함 정규식이 가장 안정적.

### 3-2. 공모희망가 밴드 (`band_low` / `band_high`)

- **출현 조건**: 모든 정정/확정 신고서에 일관 등장.
- **신호 문구 (3건 공통)**:
  > "공모희망가액인 **5,300원 ~ 6,000원**" (코스모로보틱스)
  > "희망공모가액인 **19,000원 ~ 21,500원**" (스트라드비젼)
  > "공모희망가 **12,400원 ~ 14,800원**" (피스피스스튜디오)

- **정규식**:
  ```
  /(?:공모\s*희망\s*가액?|희망\s*공모\s*가액?|공모\s*희망\s*가)\s*인?\s*([\d,]+)\s*원\s*~\s*([\d,]+)\s*원/
  ```
- **추출 규칙**: 첫 매치의 그룹 1·2 → `parsePrice_()`. `band_low < band_high` 검증.
- **함정 1**: HTML 테이블 셀이 잘못 합쳐지면 `5,3000원 ~ 6,000원` 같은 오타가 본문 상단에 등장(코스모 사례). → "공모희망가액인" 라벨이 앞에 붙은 매치만 채택해 회피.
- **함정 2**: 복수 매치 가능 (의무보유확약 절에 동일 값 재등장). 첫 매치만 사용.

## 4. 추출 불가 필드 (document.xml에서 직접 안 됨)

### 4-1. 기관 수요예측 경쟁률 (`demand_competition`)

- **이유**: 정정/확정 신고서 본문에는 수요예측 결과 표가 들어가지 않음. 별도 [투자설명서] 공시 또는 [증권발행실적보고서]에 포함되나 별도 rcept_no.
- **결정**: P1-B에서는 38.co.kr 값만 사용. DART에서는 추출 시도 X.

### 4-2. 의무보유확약 비율 (`lockup_pct`)

- **이유**: 본문에는 "의무보유 확약" 단어는 빈출하나, 비율 표(예: 3M 30% / 6M 25% / 1Y 15%)는 정정신고서 단계에서는 미공시. 발행 후 [증권발행실적보고서]에 등장.
- **결정**: P1-B에서는 38.co.kr 값만 사용. DART에서는 추출 시도 X.

**confidence 재산정 로직 (단계 4)**:
- DART rcept_no 매칭됨 + 3개 가격(`confirmed_price`/`band_low`/`band_high`) 중 추출된 것이 모두 38과 일치 → `high` ("✓ 공식 교차확인")

- DART 매칭 안 됨 (38 only) → `medium`
- DART만 있는 케이스 → `low` (실무상 거의 없음)

## 6. Apps Script 구현 노트

- **ZIP 해제**: `Utilities.unzip(blob)[0].getDataAsString('UTF-8')`
- **레이트**: 80건 × 300ms = 24초. Apps Script 6분 한도 안전.
- **DART API 한도**: 20,000건/일. 80건 × 2회 = 160건/일 → 0.8% 사용. 안전.
- **재사용 헬퍼**: `parsePrice_()` (line 481). 신규 헬퍼 불필요.
- **에러 격리**: `try/catch` per rcept_no. document.xml 실패 시 null 반환 → 기존 38 fallback 정상 동작.
