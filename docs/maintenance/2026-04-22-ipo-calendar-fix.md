## 5. P2 — 데이터 소스 처음부터 재설계 (3~4주, 12~16h)

> **사용자 지시**: "데이터 소스를 아예 처음부터 P2 플랜으로" — 38.co.kr 위 땜질이 아니라 공식 원천 기반 파이프라인으로 전면 재작성

### 5.1 원천 리서치 (P2-1, 3h)
- **DART 전자공시** (공식, 무료 API key) — 증권신고서·투자설명서·공모가·청약일정·주관사
- **KIND 한국거래소 상장공시** — 상장일·시장구분 확정치
- **38.co.kr** — 보조 소스 유지 (기관수요예측 경쟁률·유통비율 등 DART 미제공 지표)
- 각 원천의 **커버리지 매트릭스 작성** — 종목×4대 팩터×청약단계

### 5.2 아키텍처 설계 (P2-2, 2h)
- **다층 파이프라인**: `raw_dart` + `raw_kind` + `raw_38` → `normalized` → `archive` (4개 시트)
- **교차 검증 로직** — DART 우선, 38.co.kr 보조, 불일치 시 DART 우선 + 플래그 기록
- **실패 복원력** — 원천별 독립 에러 핸들링 + 실패 시 이전 캐시 유지

### 5.3 4대 팩터 재정의 (P2-3, 2h)
1. **기관 경쟁률** → "기관 수요예측 경쟁률"로 레이블 명확화, DART 증권신고서의 정정 공시 반영
2. **확정공모가 위치** → 5단계 구분 (밴드 초과/상단/중간/하단/하회), `calcGrade` 입력값 재설계
3. **의무보유확약** → DART 투자설명서의 기관별 확약 내역 합산, 기간별 세분 (3개월/6개월/1년)
4. **유통가능물량** → 상장 시점 유통가능 주식수 / 총 상장주식수 명시적 산출

### 5.4 새 스크래퍼·로더 구현 (P2-4, 4~6h)
- Apps Script 리라이트 (현 `ipo-scraper.gs` 대체)
- DART OpenAPI 호출 (corp_code 조회 + 고유번호 기반 공시 검색)
- KIND 공시 크롤링 (HTML 파싱)
- 38.co.kr 보조 필드 유지
- 트리거: 매일 오전 8시 + 오후 6시 2회 (청약 시작/종료 경계 반영)

### 5.5 프론트 교체 (P2-5, 1~2h)
- `ipo.html`의 `loadFromSheets()` → `loadFromNormalizedSheet()` 로 전환
- 컬럼 구조 변경에 따른 `val(i)` 인덱스 재매핑
- 경쟁률 레이블 "기관 수요예측 경쟁률 X:1" 표기
- 밴드 5단계 표기 UI
- JSON-LD Event 동적 주입 (청약예정 상태만)

### 5.6 아카이브·독립 URL 확장 (P2-6, 2~3h)
- `archive` 시트 별도 관리 (상장 완료 후 6개월치 유지)
- `/ipo/archive` 페이지 추가 (검색·필터)
- 종목별 독립 URL `/ipo/[종목명-slug]` — AEO 인용 가능성 ↑ (사이클 6-1 Top 20 연계)
