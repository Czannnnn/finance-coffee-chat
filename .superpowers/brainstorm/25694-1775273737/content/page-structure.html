공모주 분석 페이지 구조

기존 IPO 캘린더(ipo.html)와 새 분석 페이지의 관계를 어떻게 구성할까요?

A

별도 페이지 (ipo-analysis.html) + 캘린더에서 링크

새 독립 페이지를 만들고, 기존 캘린더의 IPO 카드에 "분석 보기" 버튼 추가.
네비게이션 바에도 메뉴 추가. 분석 페이지에서 전체 종목 테이블 + 개별 상세 펼치기.

장점

  • 캘린더 페이지 복잡도 유지
  • 분석에 집중하는 독립 뷰
  • URL 공유 가능

단점

  • 페이지 이동 필요
B

캘린더 페이지 내 탭 전환 (캘린더 ↔ 분석)

기존 ipo.html 안에 탭을 추가하여 "캘린더 뷰" / "분석 뷰"를 전환.
하나의 페이지에서 모든 IPO 정보를 관리.

장점

  • 단일 페이지로 통합
  • 페이지 이동 불필요

단점

  • ipo.html 파일이 매우 길어짐 (이미 741줄)
  • 두 기능이 커플링
C

캘린더 카드에 인라인 분석 패널 추가

기존 IPO 카드를 클릭하면 카드가 확장되면서 4대 조건 분석 결과를 보여줌.
별도 페이지 없이 기존 캘린더에 분석 기능 통합.

장점

  • 맥락 유지 (캘린더와 함께)
  • 별도 페이지 불필요

단점

  • 전체 종목 비교가 어려움
  • 확장 시 레이아웃 복잡

추천: Option A — 별도 페이지로 분석에 집중하면서, 캘린더에서 자연스럽게 연결. 파일 관리도 깔끔.
